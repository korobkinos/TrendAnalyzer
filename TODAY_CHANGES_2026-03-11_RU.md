# Изменения за 2026-03-11

## Что сделано сегодня

1. Исправлен ложный диалог «Есть несохраненные изменения» при выходе.
- Причина: в dirty-check сравнивались runtime-поля `ui_state` (`x_range`, `scale_states.y_min/y_max`), которые меняются автоматически в `Авто X/Авто Y`.
- Решение: добавлена нормализация профиля перед сравнением в `_has_unsaved_config_changes()`.

2. Переработана логика нижней таблицы под графиком.
- Теперь в таблице отображаются только активные сигналы (`enabled=True`), то есть реально выведенные на график.
- «База сигналов» остается в отдельном окне `Сигналы графика`.

3. Добавлен drag-and-drop из окна `Сигналы графика` на график.
- Реализован `SignalLibraryTable` (drag source).
- При перетаскивании сигнал автоматически включается на графике.

4. Устранены зависания/вылеты при массовом изменении видимости (например, 128 сигналов).
- В `chart.py` добавлен пакетный метод `set_signals_enabled()`.
- Массовые команды «Показать/Скрыть выбранные» переведены на batch-обновление (1 redraw вместо сотен).
- Для отключенных сигналов очищается кривая (`setData([], [])`) для снижения нагрузки.

5. Добавлено сохранение/восстановление активного набора сигналов в конфигурации.
- В `ui_state.view` сохраняется `active_signal_ids`.
- При загрузке профиля и применении конфигурации активность сигналов восстанавливается из `active_signal_ids`.

## Дополнение от 2026-03-12

1. Унифицирован прогресс-бар для сохранений (busy state).
- `Приложение -> Сохранить конфигурацию` теперь всегда показывает индикатор выполнения и корректно завершает сохранение.
- `Сигналы графика -> Сохранить конфигурацию` сохранение обернуто в единый busy-проход (`применение + сохранение`).
- Добавлен индикатор выполнения для долгих операций сохранения архива (`.trend.json/.trend.zip`) и экспорта подключения.

2. Исправлен расчет размера архива в статус-баре.
- Ранее учитывался только `archive.db`, из-за чего казалось, что размер «не растет».
- Теперь учитывается полный размер SQLite: `archive.db + archive.db-wal + archive.db-shm`.
- При доступном `ArchiveStore` используется его метод `db_size_bytes()` (тоже суммарный размер).

3. Доработана безопасность сохранения при выходе.
- В принудительном закрытии сохранение выполняется без лишнего busy-cycle (`show_busy=False`), чтобы избежать реэнтрантности в `closeEvent`.

4. Добавлен тест на размер архива с WAL/SHM.
- Новый тест: `test_archive_size_counts_wal_and_shm_parts` в `tests/test_ui_dirty_check.py`.
